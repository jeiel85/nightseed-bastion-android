# 07_MOBILE_UX_GUIDE

## Screen Format

- Primary: 720x1280 portrait reference.
- Safe UI target: 640x1136 logical design area.
- Avoid critical buttons at extreme corners.
- Support Android gesture navigation safe area.

## Main Menu

Required buttons:

1. Continue
2. New Run
3. Heroes
4. Codex
5. Settings

Secondary bottom nav:

- Achievements
- Daily Reward, if added later
- Shop, only if ethical monetization is approved
- Mailbox, only if online services are added later

## In-Run HUD

### Day HUD

- Top: Night number, Moonshards, Core HP
- Center: Map and build slots
- Bottom: Build cards
- Right: Begin Dusk button

### Dusk HUD

- Top: Omen summary
- Lane overlays: enemy icons and danger intensity
- Bottom: Bargain cards or skip button

### Night HUD

- Top: Core HP, wave progress, timer
- Left bottom: virtual joystick
- Right bottom: skill buttons
- Center: combat readability; no permanent UI clutter

## Touch Rules

- Tap build slot: inspect or place.
- Long press building: detail tooltip.
- Drag build card to slot: optional later.
- Double-tap should not be required for essential gameplay.

## Accessibility Baseline

- No tiny text below 12 logical pixels.
- Icons must have text labels in menus.
- Color cannot be the only status indicator.
- Camera shake must be adjustable.
- Flashing effects must be minimal.
